## Create Documentation

Create documentation or guidance files for the project:

1. **Identify the documentation need:**
   - What topic needs documentation?
   - Who is the audience?
   - What type of documentation is needed?

2. **Research:**
   - Check existing documentation in `docs/`, `README.md`, `CLAUDE.md`
   - Review related code for accuracy

3. **Create the documentation:**
   - Use appropriate format (Markdown for most cases)
   - Include examples and code snippets
   - Add cross-references to related docs

4. **Place in appropriate location:**
   - User-facing docs → `docs/` directory
   - Developer guidance → `.claude/` directory
   - Architecture docs → `docs/` or `ARCHITECTURE.md`

## Common Documentation Types

| Type | Purpose | Example |
|------|---------|---------|
| Setup Guide | Getting started | "How to set up the project" |
| Usage Guide | How to use features | "How to add bookmarks" |
| API Reference | Technical documentation | "Context API usage" |
| Architecture | System design | "Data flow diagram" |
| Troubleshooting | Problem solutions | "Fixing hydration errors" |

## Quality Checklist

- [ ] Clear title and purpose
- [ ] Logical structure with headings
- [ ] Working code examples
- [ ] Cross-references to related docs
- [ ] No duplicate information
